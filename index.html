Frequently Asked Questions

1. What is the difference between var, let, and const?
Answer: var: which is function scope that can be re-declared and re-assigned, hoisted with undefined. let: is block-scoped, allows changes but keeps things tidy by not letting you re-declare in the same scope. const: It also has block scope, and cannot be re-assigned.
2. What is the difference between map(), forEach(), and filter()?
Answer: map() transforms each element of an array and returns a new array, while forEach() simply iterates over the elements without returning anything. On the other hand, filter() creates a new array by selecting elements that meet a specified condition.
3. What are arrow functions, and how are they different from regular functions?
Answer: Arrow functions are a concise way to write functions using the => syntax. They differ from regular functions in that they don’t have their own this context and instead inherit this from the surrounding scope. Additionally, arrow functions don’t have an arguments object, unlike regular functions, which have their own this and arguments objects.
4. How do JavaScript Promises work?
Answer: JavaScript Promises represent the outcome of an asynchronous operation. A Promise can be pending, fulfilled with a value, or rejected with an error. Promises use then() to handle successful results, catch() to handle errors, and finally() to run code after the promise settles, regardless of the outcome.
5. How does closure work in JavaScript?
Answer: Closures in JavaScript occur when an inner function retains access to variables from its outer function even after the outer function has executed. This allows the inner function to remember and manipulate the variables of the outer scope. Closures are commonly used for data encapsulation and maintaining state in asynchronous code or event handlers.
